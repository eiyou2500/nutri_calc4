# Anti-Slop Audit

1. **このサイトを5秒見て、何のサイトか分かるか。** Yes — 先頭に計算フォームがあり、点滴・食事・経腸・ONSの入力から始まる。
2. **shadcn/uiのデフォルトテーマに見えないか。** Yes — 静的PWAでshadcn未導入。淡いコーラル、生成り背景、丸い入力欄で既存UIを再編集した。
3. **どこかで見たSaaSテンプレートに見えないか。** Yes — ランディングヒーローを削除し、計算フォームを最初に表示する実用品の画面に戻した。
4. **heroの視覚モチーフは、このサイト固有か。** Yes — ヒーロー自体を削除し、実務上不要なモチーフより入力の早さを優先した。
5. **カードを消しても情報構造が残るか。** Yes — 点滴、食事、経腸、ONS、合計、体格データの作業順序が残る。
6. **抽象コピーでごまかしていないか。** Yes — 不要な宣伝コピーを削除し、フォーム内の具体ラベル中心にした。
7. **色はターゲットの性格から来ているか。** Yes — 20代女性が日常的に使いやすい淡いコーラルと温かい生成りを、医療用途に耐えるコントラストで使った。
8. **余白に意図があるか。** Yes — 上部説明をなくして即入力できる密度にし、合計だけを少し強く見せた。
9. **モーションは状態変化を説明しているか。** Yes — hover/focusの小さな反応に限定し、常時動く装飾はない。
10. **mobileで“ただ積んだだけ”になっていないか。** Yes — 既存のスマホ入力密度を維持し、上部の不要スクロールを削除した。
11. **hover/focus/active/disabled/loading/empty stateは整っているか。** Yes — hover/active/disabled/focus-visibleをCSSで明示し、既存empty/null stateも維持した。
12. **1つでも記憶に残る細部があるか。** Yes — 丸い「栄」印とセクション見出しの小さなコーラル点を残した。
13. **逆に、不要な細部を足しすぎていないか。** Yes — ヒーローアーティファクトを削除し、装飾は入力補助に関係する最小限にした。
14. **すべてのセクションが同じテンションになっていないか。** Yes — 入力欄は軽く、合計欄は少し強く、管理欄は控えめにした。
15. **このUIからロゴを外しても、人格が残るか。** Yes — 淡い手帳風の計算シートという人格が残る。
