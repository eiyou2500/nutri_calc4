# Anti-Slop Audit

1. **このサイトを5秒見て、何のサイトか分かるか。** Yes — ヒーローで「食事・点滴・経腸・ONSを、同じ紙面で計算する」と明示した。
2. **shadcn/uiのデフォルトテーマに見えないか。** Yes — 静的PWAでshadcn未導入。OKLCHの紙面色、罫線、非対称角丸で既存テーマを再編集した。
3. **どこかで見たSaaSテンプレートに見えないか。** Yes — ダッシュボード風の抽象ヒーローではなく、病棟の栄養指示簿を想起する罫線と集計欄にした。
4. **heroの視覚モチーフは、このサイト固有か。** Yes — 入力区分が一日量に束ねられる「栄養指示簿の計算欄」をモチーフにした。
5. **カードを消しても情報構造が残るか。** Yes — 点滴、食事、経腸、ONS、合計、体格データの順に作業順序が残る。
6. **抽象コピーでごまかしていないか。** Yes — kcal、水分、電解質、糖質/インスリン比、GIRなど具体語を使用した。
7. **色はブランドの性格から来ているか。** Yes — 医療現場の落ち着きに合う青緑と紙面の生成り、警告の橙赤に絞った。
8. **余白に意図があるか。** Yes — ヒーローは説明、入力部は密度、合計部は強調という違いを持たせた。
9. **モーションは状態変化を説明しているか。** Yes — ヒーローのバーは入力が集計へ収束する説明に限定し、reduced motionにも対応した。
10. **mobileで“ただ積んだだけ”になっていないか。** Yes — 既存のモバイル入力密度を維持しつつ、ヒーローの罫線を消して読み順を優先した。
11. **hover/focus/active/disabled/loading/empty stateは整っているか。** Yes — hover/active/disabled/focus-visibleをCSSで明示し、既存empty/null stateも維持した。
12. **1つでも記憶に残る細部があるか。** Yes — 「栄」の小さな印影ロゴと、計算欄の鉛筆罫線を追加した。
13. **逆に、不要な細部を足しすぎていないか。** Yes — 装飾は罫線、集計バー、印影に限定した。
14. **すべてのセクションが同じテンションになっていないか。** Yes — ヒーロー、入力カード、合計、マスタ管理で密度と境界表現を変えた。
15. **このUIからロゴを外しても、人格が残るか。** Yes — 紙面罫線と栄養計算欄の編集思想が残る。
